## Example: 2 つのプロジェクトの比較

**プロジェクト A の NPV：**

$$
NPV_A = -1,000 + \frac{300}{1.10} + \frac{300}{(1.10)^2} + \frac{300}{(1.10)^3} + \frac{300}{(1.10)^4} + \frac{300}{(1.10)^5}
$$

$$
NPV_A = -1,000 + 273 + 248 + 225 + 205 + 186 = +137 \text{万円}
$$

**プロジェクト B の NPV：**

$$
NPV_B = -800 + \frac{250}{1.10} + \frac{250}{(1.10)^2} + \frac{250}{(1.10)^3} + \frac{250}{(1.10)^4} + \frac{250}{(1.10)^5}
$$

$$
NPV_B = -800 + 227 + 207 + 188 + 171 + 155 = +148 \text{万円}
$$

**結論：**

プロジェクト B の方が NPV が高い（148 万円 > 137 万円）ので、 **プロジェクト B を選ぶべき** です。

初期投資はプロジェクト A の方が大きく、一見すると規模の大きい A の方が魅力的に見えますが、 **価値が多く増えるのは B の方** です。判断基準は「初期投資の大きさ」ではなく、 **「最終的に価値をいくら増やすか（= NPV）」** であることに注意してください。

## Example: リスクを考慮しない間違った比較

### ❌ 間違った比較：同じ割引率（10%）を使う

**プロジェクト A の NPV（割引率 10%）：**

$$
NPV_A = -1,000 + \frac{450}{1.10} + \frac{450}{(1.10)^2} + \frac{450}{(1.10)^3} + \frac{450}{(1.10)^4} + \frac{450}{(1.10)^5}
$$

$$
NPV_A = -1,000 + 409 + 372 + 338 + 307 + 280 = +706 \text{万円}
$$

**プロジェクト B の NPV（割引率 10%）：**

$$
NPV_B = -1,000 + \frac{500}{1.10} + \frac{500}{(1.10)^2} + \frac{500}{(1.10)^3} + \frac{500}{(1.10)^4} + \frac{500}{(1.10)^5}
$$

$$
NPV_B = -1,000 + 455 + 413 + 376 + 342 + 311 = +897 \text{万円}
$$

**❌ 間違った結論：**

「プロジェクト B の方が NPV が高い（897 万円 > 706 万円）ので、プロジェクト B を選ぼう！」

### ✅ 正しい比較：リスクに応じて割引率を変える

プロジェクト A は **低リスク** なので、割引率は **8%** 。

プロジェクト B は **高リスク** なので、割引率は **20%** 。

（Chapter 2 で学んだとおり、 **リスクが高いプロジェクトには高い割引率を使う** のが原則です。「不確実なリターンは、確実なリターンより低く評価されるべき」だからです）

**プロジェクト A の NPV（割引率 8%）：**

$$
NPV_A = -1,000 + \frac{450}{1.08} + \frac{450}{(1.08)^2} + \frac{450}{(1.08)^3} + \frac{450}{(1.08)^4} + \frac{450}{(1.08)^5}
$$

$$
NPV_A = -1,000 + 417 + 386 + 357 + 331 + 306 = +797 \text{万円}
$$

**プロジェクト B の NPV（割引率 20%）：**

$$
NPV_B = -1,000 + \frac{500}{1.20} + \frac{500}{(1.20)^2} + \frac{500}{(1.20)^3} + \frac{500}{(1.20)^4} + \frac{500}{(1.20)^5}
$$

$$
NPV_B = -1,000 + 417 + 347 + 289 + 241 + 201 = +495 \text{万円}
$$

**✅ 正しい結論：**

リスクに応じた割引率で計算しなおすと、NPV のランキングは **完全に逆転** しました。

| 比較方法                      | プロジェクト A | プロジェクト B | 結論         |
| ----------------------------- | -------------: | -------------: | :----------- |
| ❌ 同じ割引率（10%）で比較    |      +706 万円 |  **+897 万円** | 「B を選ぶ」 |
| ✅ リスクに応じた割引率で比較 |  **+797 万円** |      +495 万円 | 「A を選ぶ」 |

もしリスクを無視して同じ割引率で判断していたら、 **本当は価値の増加が小さい B を選んでしまい、より価値を増やす A を逃す** ところでした。

なぜリスクに応じた割引率の方が **公平な比較に近い** かというと、 **「不確実なリターンは確実なリターンより低く評価する」というあたりまえの感覚を、数字に反映させているから** です。同じ「年 500 万円」の CF でも、 **確実に入る 500 万円** と **入るかどうか分からない 500 万円** は価値が違う——その差を表現するのが割引率の役割です。

なお、ここで使った 8% / 20% という具体的な数字は説明用の仮置きであり、実務では CAPM や build-up method、社内のハードルレート対応表などから導きます（本文 3.2 節のコラム参照）。

## Example: Payback Period の罠

**プロジェクト A の Payback Period ＝ 1.75 年** （ $1 + 300/400$ ）

**プロジェクト B の Payback Period ＝ 3.17 年** （ $3 + 100/600$ ）

**❌ Payback Period で判断すると：**

「2 年以内」ルールでは A だけが承認され、B は却下されます。「A を選ぶ」が結論。

**✅ NPV（割引率 10%）で比較すると：**

$$
NPV_A = -1,000 + \frac{700}{1.10} + \frac{400}{(1.10)^2} + 0 + 0 + 0 = -1,000 + 636 + 331 = -33 \text{万円}
$$

$$
NPV_B = -1,000 + \frac{250}{1.10} + \frac{250}{(1.10)^2} + \frac{400}{(1.10)^3} + \frac{600}{(1.10)^4} + \frac{600}{(1.10)^5} = -1,000 + 227 + 207 + 301 + 410 + 373 = +518 \text{万円}
$$

**比較表：**

| 指標           |           プロジェクト A |            プロジェクト B | Payback ルールの判定   |
| -------------- | -----------------------: | ------------------------: | :--------------------- |
| Payback Period |              **1.75 年** |                   3.17 年 | **A を選ぶ・B を却下** |
| NPV @ 10%      | **−33 万円（価値破壊）** | **+518 万円（価値創造）** | **完全に間違い**       |

**結論：**

Payback Period のルールに従うと、 **本当は価値を減らす A を承認し、+518 万円もの価値を生む B を却下してしまう** ことになります。差し引き **551 万円ぶん** の意思決定ミスです。

なぜこんなことが起こるかというと、 Payback Period は **「回収できた後のキャッシュフローを完全に無視する」** からです：

- A：Payback 後（Y3〜Y5）は CF がゼロなので、回収後は何も生み出さない。 **短期で死ぬプロジェクト** 。
- B：Payback 後（Y4〜Y5）に毎年 600 万円を生み続ける。 **長く価値を生み続けるプロジェクト** 。

Payback Period は「回収できたかどうか」しか見ないので、 **回収後に生まれる本当の価値が指標に乗ってこない** のです。

**教訓：**

Payback Period を **「最低限の足切りライン（資金繰りチェック）」** として使うのは構いません。しかし **「Payback が短いほど良いプロジェクトだ」という発想で投資判断をしてはいけません** 。最終判断は必ず **NPV で行う** こと。
