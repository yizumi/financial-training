# 3.6 ケース演習 解答と詳細解説：開発プロジェクト A vs B

> 本ファイルは Chapter 3 の 3.6 ケース演習（[chapter-3.md](./chapter-3.md) 参照）の **解答と詳細解説** です。当日のワークショップ後に配布する想定で作成しています。

## ケース概要（再掲）

あなたは SaaS 企業の CTO です。 **どちらか一方しか実行できない** 2 つの開発プロジェクトについて、最適な投資判断を下してください。

- **プロジェクト A：AI チャットボット** 開発
  - キャッシュフロー：-3,000 / +2,000 / +1,200 / +500 / +200 / +100 （万円、Y0〜Y5）
- **プロジェクト B：パフォーマンス最適化** （基盤刷新）
  - キャッシュフロー：-2,000 / +400 / +600 / +900 / +1,100 / +1,100 （万円、Y0〜Y5）
- **割引率：10%（WACC）**

## 分析手順

### ステップ 1：NPV を計算

**プロジェクト A の NPV：**

$$
NPV_A = -3,000 + \frac{2,000}{1.10} + \frac{1,200}{(1.10)^2} + \frac{500}{(1.10)^3} + \frac{200}{(1.10)^4} + \frac{100}{(1.10)^5}
$$

$$
NPV_A = -3,000 + 1,818 + 992 + 376 + 137 + 62 = +385 \text{万円}
$$

**プロジェクト B の NPV：**

$$
NPV_B = -2,000 + \frac{400}{1.10} + \frac{600}{(1.10)^2} + \frac{900}{(1.10)^3} + \frac{1,100}{(1.10)^4} + \frac{1,100}{(1.10)^5}
$$

$$
NPV_B = -2,000 + 364 + 496 + 676 + 751 + 683 = +970 \text{万円}
$$

**NPV で比較：** プロジェクト B（+970 万円）> プロジェクト A（+385 万円）。 **B の方が約 2.5 倍の価値創造** 。

### ステップ 2：IRR の確認（事前計算済み）

| プロジェクト | IRR       | Excel での計算式                           |
| ------------ | --------- | ------------------------------------------ |
| A            | **18.2%** | `=IRR({-3000, 2000, 1200, 500, 200, 100})` |
| B            | **24.5%** | `=IRR({-2000, 400, 600, 900, 1100, 1100})` |

**IRR で比較：** プロジェクト B（24.5%）> プロジェクト A（18.2%）。 **B が 6.3 ポイント高い** 。

両方とも WACC（10%）を上回っているので、 **単独 accept/reject の判断ならどちらも投資価値あり** です。

### ステップ 3：Payback Period を計算

**プロジェクト A：**

| 年度 | CF（万円） | 累積 CF（万円） |
| ---: | ---------: | --------------: |
|    0 |     -3,000 |          -3,000 |
|    1 |     +2,000 |          -1,000 |
|    2 |     +1,200 |            +200 |

$$
\text{Payback}_A = 1 + \frac{1,000}{1,200} \approx 1.83 \text{年}
$$

**プロジェクト B：**

| 年度 | CF（万円） | 累積 CF（万円） |
| ---: | ---------: | --------------: |
|    0 |     -2,000 |          -2,000 |
|    1 |       +400 |          -1,600 |
|    2 |       +600 |          -1,000 |
|    3 |       +900 |            -100 |
|    4 |     +1,100 |          +1,000 |

$$
\text{Payback}_B = 3 + \frac{100}{1,100} \approx 3.09 \text{年}
$$

**Payback Period で比較：** プロジェクト A（1.83 年）< プロジェクト B（3.09 年）。 **A の方が 1.26 年も早く回収** 。

## 結論：A と B、どちらを選ぶべきか？

3 つの指標で結果が **割れました** 。

| 指標           | プロジェクト A | プロジェクト B | 優位 |
| -------------- | -------------: | -------------: | :--: |
| NPV            |      +385 万円 |  **+970 万円** |  B   |
| IRR            |          18.2% |      **24.5%** |  B   |
| Payback Period |    **1.83 年** |        3.09 年 |  A   |

**Payback Period では A が圧倒的に優位** である一方、 **NPV・IRR では B が優位** という、典型的な「指標が対立する」ケースです。

### 教科書的な答え：B を選ぶ

ここまで学んできた原則に従えば、答えは明確です：

> **「投資判断の最終基準は NPV である。」**

NPV は「企業価値を何円増やすか」を直接測ります。プロジェクト B は **A より 585 万円多くの価値を生む** ので、合理的な投資家・経営者は B を選びます。

加えて、 **Payback Period の根本的な弱点は「回収期間後の CF を無視すること」** です（3.4 で扱った通り）。今回の例ではまさにそれが効いています：

- **A** ：前倒しで早く回収するが、コモディティ化で **後半は急速に価値を失う** 。Y3 以降の合計 CF はわずか 800 万円。
- **B** ：立ち上がりは遅いが、Y4・Y5 で毎年 1,100 万円ずつ生み続ける。 **Payback 後の CF（合計 2,200 万円）が大きい** 。

Payback Period が「A 優位」と言うのは、 **「A が短期間しか価値を生まない」という弱点をむしろ反映してしまっている** のです。NPV はこれを正しく評価して B を選びます。

### では、A を選ぶべき場面はあるのか？

NPV で B が勝つにもかかわらず、 **以下のような状況では A が現実解になる** ことがあります：

1. **会社のキャッシュが逼迫している（流動性リスク）**
   - スタートアップやシリーズ A 直後など、 **次の資金調達まで現金を絶対に切らせない** という制約があるとき
   - 1.83 年で投資を全額回収できる A は、 **資金繰りリスクが極めて低い**
   - 一方、B は 3 年強の間、3,000 万円相当の資金が「塩漬け」になる

2. **将来の不確実性が想定以上に高い**
   - WACC = 10% にリスクは織り込まれているが、 **「3 年後にこの市場が存在しているか分からない」** ような高不確実環境では、後ろ倒しの CF はさらに割り引いて考えるべき
   - B の Y4・Y5 の CF（合計 2,200 万円）が予測通り入る保証は薄くなる

3. **次の有望な投資機会がある**
   - 早く回収したキャッシュを **次の高 NPV プロジェクトに投じられる** 場合、A の流動性メリットが効く
   - これは IRR の「再投資仮定」とは別で、 **実際に高利回りの再投資先がある** ケース

4. **市場差別化・ブランド戦略上の意義**
   - AI チャットボットは **対外的に「先進企業」のシグナル** を発する（採用・営業・PR に効く）
   - 数字に現れない戦略的価値

### 意思決定フローチャート

```
NPV B > NPV A？
  ↓ Yes
キャッシュは潤沢か（最低 18 ヶ月分のランウェイがあるか）？
  ├─ Yes →【B を選ぶ】 ← 教科書的な正解
  └─ No  → A の Payback 優位は流動性リスク低減に役立つか？
            ├─ 役立つ →【A を選ぶ】流動性確保が最優先
            └─ 役立たない →【B を選ぶ】（資金調達で対処）
```

### 定性的要因の考慮

**プロジェクト A の追加メリット：**

- 市場での差別化要因になる（リリース時点では競合にない機能）
- 顧客満足度の向上、ブランドイメージの向上
- 1.83 年でキャッシュ回収 → **流動性リスクが極小**

**プロジェクト A の追加リスク：**

- AI 技術のコモディティ化が **想定より速く進む** と、Y2 以降の CF も下振れする
- 差別化が消えた後、ユーザーは「あって当然」の機能と捉え、 **解約抑止効果は限定的**

**プロジェクト B の追加メリット：**

- 既存顧客の解約率が下がる（継続率向上 → LTV 拡大）
- チーム全体の開発速度が向上（技術的負債の削減）
- スケーラビリティの向上（将来の成長に対応）
- **競合に簡単にコピーされない持続的な競争優位**

**プロジェクト B の追加リスク：**

- 移行期間中（Y1〜Y2）は **本体の現金収支が痛む**
- 大規模リファクタリングは **遅延・予算超過のリスク** が高い

### 最終判断

| 状況                                     | 推奨         |
| ---------------------------------------- | ------------ |
| キャッシュが潤沢で、長期視点で投資できる | **B を選ぶ** |
| キャッシュ逼迫・短期の現金回収が経営課題 | **A を選ぶ** |
| 不確実性が高く、3 年後の予測が立てにくい | **A を選ぶ** |
| 戦略上、技術基盤の競争優位を最優先したい | **B を選ぶ** |

**多くの SaaS 企業（成長期で流動性に余裕があり、長期視点で運営できる）にとっては、 B が正解です** 。NPV が示す通り、 **企業価値を 2.5 倍多く増やす** からです。

ただし、本ケースが教えてくれる重要な点は：

> **「指標がぶつかったときこそ、自社のキャッシュ状況・戦略・リスク許容度を再確認するチャンスである。」**
>
> **「同じ財務データでも、置かれた状況によって最適解は変わる。」**

NPV はあくまで **「価値創造額のベストエスティメート」** であり、それを **どう取りに行くか** は経営判断です。
